# 猕猴脑电生理数据分析软件 UI需求规格说明书

## 1. 项目概述

### 1.1 软件名称
NeuroPrime - 猕猴脑电生理数据分析平台

### 1.2 开发目标
开发一款基于PyQt5的专业电生理数据分析软件，支持猕猴脑电信号的完整分析流程，包括数据加载、预处理、Spike检测与排序、LFP分析、行为对齐、解码分析等功能。

### 1.3 运行环境
- **运行方式**: Python脚本运行（开源项目）
- **UI框架**: PyQt5
- **可视化库**: PyQtGraph
- **数据格式**: HDF5
- **工作模式**: 手动逐步执行

**电信号数据预处理流程：**

| 步骤 | 操作 | 输出 |
|------|------|------|
| 1. 读取原始数据 | 根据文件格式读取数据 | 原始信号数组 |
| 2. 格式转换 | 转换为标准数据结构 | 标准信号格式 |
| 3. 信号预处理 | 滤波、重参考 | 滤波后信号 |
| 4. Spike检测 | 阈值检测提取Spike | Spike时间戳、波形 |
| 5. LFP提取 | 低通滤波提取LFP | LFP信号 |
| 6. 保存到HDF5 | 按标准结构保存 | HDF5文件 |

**行为数据预处理流程：**

| 步骤 | 操作 | 输出 |
|------|------|------|
| 1. 读取原始数据 | 读取行为事件文件 | 事件时间、标签 |
| 2. 格式转换 | 转换为标准事件结构 | 标准事件格式 |
| 3. 事件对齐 | 与电信号时间对齐 | 对齐后事件 |
| 4. 试次划分 | 根据事件划分试次 | 试次信息 |
| 5. 保存到HDF5 | 按标准结构保存 | HDF5文件 |

**错误处理机制：**
- 预处理失败时弹出对话框，让用户选择：
  - 【终止】停止预处理，显示错误信息
  - 【跳过】跳过失败步骤，继续后续步骤
  - 【重试】重新执行失败步骤

**导航栏更新：**
- 预处理完成后，新数据节点自动添加到左侧导航栏
- 默认不自动展开，保持界面简洁

**数据内容说明**:

每个试验对应一个独立的HDF5文件 `{trial_name}.h5`，内部结构如下：

**Spike数据节点展开后** (对应HDF5文件内 `/spikes` 组):
- 📄 时间戳 → `/spikes/spike_times`
- 📄 波形 → `/spikes/spike_waveforms`
- 📄 聚类标签 → `/spikes/cluster_labels`

**电信号数据节点** (对应HDF5文件内 `/signals` 组):
- ⚡ Spike数据 → 包含Spike时间戳、波形、聚类结果（链接到/spikes）
- 📄 LFP信号 → `/signals/lfp_data`

**行为数据节点** (对应HDF5文件内 `/behavior` 组):
- 📄 事件 → `/behavior/events`

## 7. 统一数据接口设计

### 7.1 架构设计
```
┌─────────────────────────────────────────────────────────────┐
│                    统一数据接口层                             │
├─────────────────────────────────────────────────────────────┤
│  原始数据 → 预处理 → 标准数据结构 → 算法处理 → 标准结果结构    │
│  (各种格式)   (格式转换)   (HDF5)    (内置/自定义)  (可视化)  │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 标准数据结构（HDF5）

#### 7.2.1 电信号数据
```
/signals
  ├── raw/              # 原始信号
  │   ├── data          # [n_channels, n_samples] 数组
  │   ├── fs            # 采样率 (Hz)
  │   ├── channel_names # 通道名称列表
  │   └── unit          # 单位 (如 'uV')
  ├── filtered/         # 滤波后信号（动态生成）
  │   └── ...
  └── lfp/              # LFP信号（动态生成）
      └── ...
```

#### 7.2.2 Spike数据
```
/spikes
  ├── times             # Spike时间戳 [n_spikes]
  ├── waveforms         # Spike波形 [n_spikes, n_samples]
  ├── channels          # Spike所在通道 [n_spikes]
  ├── labels            # 聚类标签 [n_spikes]
  └── units             # 单元信息
      ├── unit_ids      # 单元ID列表
      └── unit_info     # 单元元数据
```

#### 7.2.3 行为数据
```
/behavior
  ├── events
  │   ├── times         # 事件时间 [n_events]
  │   ├── labels        # 事件标签 [n_events]
  │   └── types         # 事件类型 [n_events]
  ├── trials            # 试次信息
  │   ├── start_times   # 试次开始时间
  │   ├── end_times     # 试次结束时间
  │   └── conditions    # 试次条件
  └── stimuli           # 刺激信息
      ├── directions    # 刺激方向
      ├── types         # 刺激类型
      └── parameters    # 刺激参数
```

#### 7.2.4 元数据
```
/metadata
  ├── experiment
  │   ├── name          # 实验名称
  │   ├── date          # 实验日期
  │   └── description   # 实验描述
  ├── subject
  │   ├── id            # 被试ID
  │   ├── species       # 物种
  │   └── info          # 其他信息
  └── processing
      ├── history       # 处理历史记录
      └── parameters    # 处理参数
```

### 7.3 算法接口规范

#### 7.3.1 输入接口
所有算法接收标准输入对象：
```python
class AlgorithmInput:
    signals: dict      # 电信号数据
    spikes: dict       # Spike数据
    behavior: dict     # 行为数据
    metadata: dict     # 元数据
    parameters: dict   # 用户配置参数
```

#### 7.3.2 输出接口
所有算法返回标准输出对象：
```python
class AlgorithmOutput:
    results: dict      # 分析结果数据
    figures: list      # 图表配置列表
    success: bool      # 执行状态
    message: str       # 状态信息
```

#### 7.3.3 算法基类
```python
class BaseAlgorithm:
    """算法基类，所有内置和自定义算法必须继承"""
    
    name: str           # 算法名称
    description: str    # 算法描述
    parameters: dict    # 参数定义
    
    def validate_input(self, input_data: AlgorithmInput) -> bool:
        """验证输入数据是否符合要求"""
        pass
    
    def run(self, input_data: AlgorithmInput) -> AlgorithmOutput:
        """执行算法"""
        pass
    
    def get_default_parameters(self) -> dict:
        """获取默认参数"""
        pass
```

### 7.4 接口测试功能
- **输入验证**: 检查算法是否能正确处理标准输入
- **输出验证**: 检查算法输出是否符合标准格式
- **依赖检查**: 检查算法依赖的库是否已安装
- **性能测试**: 评估算法执行时间和内存占用

### 7.5 交互流程
1. 用户点击RibbonBar上的功能按钮
2. 右侧主区域**左侧参数配置面板**显示对应算法的参数配置
3. 用户在参数面板配置参数，点击**分析按钮**
4. 软件将数据封装为标准输入对象，调用算法
5. 算法返回标准输出对象
6. 右侧主区域**右侧可视化区域**根据输出渲染图表
