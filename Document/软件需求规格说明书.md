## 1. 引言

### 1.1 编写目的
本文档详细描述了NeuroPrime猕猴脑电生理数据分析软件的功能需求、架构设计和系统规格，为软件开发、测试和维护提供依据。

### 1.2 项目背景
NeuroPrime是一款面向神经科学研究的专业电生理数据分析平台，专注于猕猴脑电信号的处理、分析和可视化。软件采用模块化架构设计，支持灵活的算法扩展和自定义功能。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| LFP | Local Field Potential，局部场电位 |
| Spike | 动作电位，神经元发放的脉冲信号 |
| PSTH | Peri-Stimulus Time Histogram，刺激前后时间直方图 |
| HDF5 | Hierarchical Data Format 5，层次化数据格式 |
| NS3 | Blackrock神经信号数据文件格式 |
| NEV | Blackrock神经事件数据文件格式 |
| MBM | 行为数据文件格式 |

## 2. 总体描述

### 2.1 软件概述
NeuroPrime是一款基于Python和PyQt6开发的桌面应用程序，提供完整的猕猴脑电生理数据分析流程，包括数据导入、预处理、Spike检测与排序、LFP分析、行为对齐、解码分析等功能。

### 2.2 运行环境

#### 2.2.1 硬件要求
- **CPU**: Intel Core i5 或同等性能以上
- **内存**: 8GB RAM 以上（推荐16GB）
- **硬盘**: 100GB 可用空间（取决于数据量）
- **显卡**: 支持OpenGL的显卡（用于可视化）

#### 2.2.2 软件环境
- **操作系统**: Windows 10/11, macOS 10.15+, Linux Ubuntu 18.04+
- **Python**: 3.8 - 3.11
- **主要依赖库**:
  - PyQt6 >= 6.4.0 (GUI框架)
  - NumPy >= 1.21.0 (数值计算)
  - SciPy >= 1.7.0 (科学计算)
  - Matplotlib >= 3.5.0 (数据可视化)
  - h5py >= 3.6.0 (HDF5文件操作)
  - scikit-learn >= 1.0.0 (机器学习)

### 2.3 用户特征
- 神经科学研究者
- 电生理实验技术人员
- 神经信息学研究人员
- 具备基础神经科学知识和数据分析能力

## 4. 功能需求

### 4.1 工程管理功能

#### 4.1.1 新建工程
- **功能编号**: F-001
- **功能描述**: 创建新的分析工程，建立工程目录结构
- **输入**: 工程名称、工程路径、描述信息
- **输出**: 工程文件夹、project.json配置文件
- **工程结构**:
  ```
  {project_name}/
  ├── project.json          # 工程配置
  ├── Document/             # 文档目录
  ├── Backup/               # 备份目录
  ├── src/                  # 源代码目录
  └── data/                 # 数据目录
      ├── raw/              # 原始数据
      └── processed/        # 处理后数据
  ```

#### 4.1.2 打开工程
- **功能编号**: F-002
- **功能描述**: 打开已有工程，加载project.json配置
- **输入**: 工程文件夹路径
- **输出**: 加载的工程配置和数据索引

#### 4.1.3 导入试验数据
- **功能编号**: F-003
- **功能描述**: 导入配套数据文件（.ns3/.nev/.mbm）
- **输入**: 数据文件路径
- **处理流程**:
  1. 解析原始数据文件
  2. 转换为HDF5格式存储
  3. 更新工程配置
  4. 建立数据索引

#### 4.1.4 批量导入
- **功能编号**: F-004
- **功能描述**: 批量导入文件夹中的所有试验数据
- **输入**: 文件夹路径
- **处理**: 自动识别并导入所有配套文件组

### 4.2 数据浏览功能

#### 4.2.1 数据导航树
- **功能编号**: F-005
- **功能描述**: 树形结构显示工程中的所有试验和数据项
- **显示内容**:
  - 试验列表（可展开/折叠）
  - 数据类型分类（LFP数据、Spike数据、行为数据）
  - 数据项详情（双击添加至分析列表）

#### 4.2.2 数据预览
- **功能编号**: F-006
- **功能描述**: 预览选中数据的基本信息和统计
- **显示内容**:
  - 采样率、通道数、时长
  - 数据范围、均值、标准差
  - 波形预览图

### 4.3 算法分析功能

#### 4.3.1 Spike检测
- **功能编号**: F-007
- **算法名称**: SpikeDetectionThreshold
- **功能描述**: 基于阈值的Spike检测
- **输入**: LFP信号数据
- **输出**: Spike时间戳、波形、通道信息
- **可调参数**:
  - 阈值系数（标准差的倍数）
  - 滤波器截止频率（低通/高通）
  - 最小间隔时间

#### 4.3.2 Spike排序
- **功能编号**: F-008
- **算法名称**: SpikeSortingPCA
- **功能描述**: 基于PCA的Spike波形聚类
- **输入**: Spike波形数据
- **输出**: 分类标签、聚类结果
- **可调参数**:
  - 主成分数量
  - 聚类数量

#### 4.3.3 LFP功率谱分析
- **功能编号**: F-009
- **算法名称**: LFPPowerSpectrum
- **功能描述**: 计算LFP信号的功率谱密度
- **输入**: LFP信号数据
- **输出**: 功率谱、频率轴
- **可调参数**:
  - 频率范围（最低/最高频率）
  - FFT点数

#### 4.3.4 LFP时频图分析
- **功能编号**: F-010
- **算法名称**: LFPSpectrogram
- **功能描述**: 计算LFP信号的时频分布
- **输入**: LFP信号数据
- **输出**: 时频图、时间轴、频率轴
- **可调参数**:
  - 窗函数类型
  - 每段样本数
  - 重叠样本数
  - 通道索引

#### 4.3.5 PSTH分析
- **功能编号**: F-011
- **算法名称**: PSTHAnalysis
- **功能描述**: 刺激前后时间直方图分析
- **输入**: Spike数据、事件标记
- **输出**: PSTH直方图

#### 4.3.6 栅格图分析
- **功能编号**: F-012
- **算法名称**: RasterPlotAnalysis
- **功能描述**: 生成Spike栅格图
- **输入**: Spike时间戳、试次信息
- **输出**: 栅格图

#### 4.3.7 调谐曲线分析
- **功能编号**: F-013
- **算法名称**: TuningCurveAnalysis
- **功能描述**: 神经元的调谐特性分析
- **输入**: Spike数据、刺激参数
- **输出**: 调谐曲线

#### 4.3.8 LDA解码
- **功能编号**: F-014
- **算法名称**: LDADecoder
- **功能描述**: 线性判别分析解码
- **输入**: 神经信号特征、行为标签
- **输出**: 解码准确率、分类结果

### 4.4 时间对齐功能

#### 4.4.1 时间范围配置
- **功能编号**: F-015
- **功能描述**: 配置多组数据的时间对齐
- **功能特性**:
  - 读取各数据文件的真实时间范围
  - 设置全局时间窗口
  - 应用到所有数据或单独配置
  - 最长时间自动对齐

### 4.5 可视化功能

#### 4.5.1 图表类型
- **功能编号**: F-016
- **支持的图表**:
  - 信号波形图
  - 原始信号多通道显示
  - Spike栅格图
  - PSTH直方图
  - 调谐曲线图
  - 功率谱图
  - 时频图

#### 4.5.2 图表操作
- **功能编号**: F-017
- **功能特性**:
  - 多标签页显示
  - 标签页关闭（点击X按钮）
  - 图表刷新
  - 数据导出
  - 关闭所有图表

### 4.6 自定义算法功能

#### 4.6.1 算法扩展接口
- **功能编号**: F-018
- **功能描述**: 支持用户自定义算法
- **实现方式**:
  - 将算法脚本放入 `custom_algorithms/` 目录
  - 继承 `BaseAlgorithm` 基类
  - 实现必需的方法接口
  - 自动加载并显示在算法选择列表

#### 4.6.2 示例算法
- **功能编号**: F-019
- **算法名称**: ViewRawData
- **功能描述**: 查看原始信号数据
- **可调参数**:
  - 通道索引
  - 起始时间
  - 显示时长
  - 显示网格
  - 线条颜色

## 5. 数据需求

### 5.1 数据格式

#### 5.1.1 原始数据格式
| 格式 | 描述 | 内容 |
|------|------|------|
| .ns3 | 神经信号数据 | 连续宽频信号（LFP + Spike） |
| .nev | 神经事件数据 | Spike事件时间戳和波形 |
| .mbm | 行为数据 | 实验参数、刺激信息、行为标记 |

#### 5.1.2 内部存储格式
- **格式**: HDF5 (.h5)
- **结构**:
  ```
  /
  ├── lfp/
  │   ├── data          [通道数 × 样本数]  float32
  │   ├── sampling_rate scalar
  │   └── time_range    [start, end]
  ├── spike/
  │   ├── times         [n_spikes]  float64
  │   ├── waveforms     [n_spikes × 波形长度]  float32
  │   └── elec_ids      [n_spikes]  int32
  ├── behavior/
  │   ├── events        结构化数组
  │   └── trial_info    结构化数组
  └── metadata/
      ├── experiment_name
      ├── creation_time
      └── source_files
  ```

## 6. 接口需求

### 6.1 算法接口

#### 6.1.1 算法基类接口
```python
class BaseAlgorithm(ABC):
    """算法抽象基类"""
    
    @abstractmethod
    def get_parameters_schema(self) -> List[AlgorithmParameter]:
        """获取参数配置模式"""
        pass
    
    @abstractmethod
    def validate_input(self, input_data: AlgorithmInput) -> bool:
        """验证输入数据"""
        pass
    
    @abstractmethod
    def run(self, input_data: AlgorithmInput, 
            parameters: Dict) -> AlgorithmOutput:
        """运行算法"""
        pass
```

#### 6.1.2 算法输入数据结构
```python
@dataclass
class AlgorithmInput:
    lfp_data: Optional[np.ndarray]      # [通道数 × 样本数]
    spike_times: Optional[np.ndarray]
    spike_waveforms: Optional[np.ndarray]
    trial_info: Optional[List[Dict]]
    events: Optional[List[Dict]]
    sampling_rate: float = 2000.0
    time_range: Optional[tuple] = None
```

#### 6.1.3 算法输出数据结构
```python
@dataclass
class AlgorithmOutput:
    data: Dict[str, np.ndarray]         # 分析结果数据
    statistics: Dict[str, float]        # 统计信息
    plot_config: Dict[str, Any]         # 图表配置
    export_data: Dict[str, np.ndarray]  # 导出数据
    metadata: Dict[str, Any]            # 元数据
    execution_time: float = 0.0
    success: bool = False
    error_message: str = ""
```

### 6.2 UI接口

#### 6.2.1 信号-槽机制
- 使用PyQt6的信号-槽机制进行模块间通信
- 主要信号:
  - `data_selected`: 数据选择改变
  - `algorithm_changed`: 算法选择改变
  - `run_analysis_requested`: 请求运行分析
  - `visualization_updated`: 可视化更新
