# HDF5文件结构分析

## 1. HDF5文件结构概述

HDF5（Hierarchical Data Format version 5）是一种用于存储和组织大型数据集的文件格式。在NeuroPrime项目中，HDF5文件用于存储神经电生理数据，包括Spike数据、LFP数据、行为数据和元数据。

## 2. 文件结构详细描述

### 2.1 顶层组结构

```
FC_Grating_012.h5/
├── behavior/           # 行为数据组
├── metadata/           # 元数据组
├── signals/            # 信号数据组（LFP）
└── spikes/             # Spike数据组
```

### 2.2 behavior组

**描述**：存储行为相关的数据，包括事件和试次信息。

| 数据集名称 | 形状 | 数据类型 | 描述 |
|-----------|------|----------|------|
| `events`  | (306,) | 复合数据类型 | 行为事件数据，包含时间戳、事件类型和值 |
| `trials`  | (61,)  | 复合数据类型 | 试次信息，包含试次编号、开始时间、结束时间、刺激条件和是否中止 |

**`events`数据集字段**：
- `timestamp`: 事件时间戳（秒）
- `event_type`: 事件类型（整数）
- `value`: 事件值（整数）

**`trials`数据集字段**：
- `trial_num`: 试次编号
- `start_time`: 试次开始时间（秒）
- `end_time`: 试次结束时间（秒）
- `stim_cnd`: 刺激条件
- `aborted`: 是否中止（1=是，0=否）

### 2.3 metadata组

**描述**：存储元数据信息。

| 子组/数据集 | 描述 |
|-------------|------|
| `source_files` | 源文件信息 |

### 2.4 signals组

**描述**：存储LFP（局部场电位）数据。

| 数据集名称 | 形状 | 数据类型 | 描述 |
|-----------|------|----------|------|
| `channel_info` | (101,) | 复合数据类型 | 通道信息，包含通道ID、电极ID、电极标签等 |
| `lfp_data` | (101, 168511) | float32 | LFP数据，维度为[通道数, 采样点数] |

**`channel_info`数据集字段**：
- `channel_id`: 通道ID
- `electrode_id`: 电极ID
- `electrode_label`: 电极标签
- `connector`: 连接器
- `pin`: 引脚
- `unit`: 单位（微伏）
- `conversion_factor`: 转换因子

### 2.5 spikes组

**描述**：存储Spike（动作电位）数据。

| 数据集名称 | 形状 | 数据类型 | 描述 |
|-----------|------|----------|------|
| `channel_info` | (144,) | 复合数据类型 | 通道信息，包含电极ID、连接器、引脚等 |
| `spike_elec_ids` | (141311,) | int64 | Spike对应的电极ID |
| `spike_times` | (141311,) | float64 | Spike时间戳（秒） |
| `spike_units` | (141311,) | int64 | Spike对应的单元ID |
| `spike_waveforms` | (141311, 48) | float32 | Spike波形数据，维度为[Spike数, 采样点数] |

**`channel_info`数据集字段**：
- `elec_id`: 电极ID
- `connector`: 连接器
- `pin`: 引脚
- `scale_nv`: 缩放因子（纳伏）
- `high_threshold_uv`: 高阈值（微伏）
- `low_threshold_uv`: 低阈值（微伏）
- `num_unit`: 单元数量

## 5. 数据流程

1. **数据采集**：通过神经记录系统采集原始神经电生理数据
2. **数据预处理**：将原始数据转换为HDF5格式，包括：
   - 提取Spike事件和波形
   - 提取LFP信号
   - 记录行为事件和试次信息
   - 添加元数据
3. **数据存储**：将处理后的数据组织到HDF5文件的各个组中
4. **数据分析**：通过NeuroPrime应用程序进行分析，如PSTH分析、栅格图等

## 6. 数据访问方式

### 6.1 使用h5py库访问

```python
import h5py

# 打开HDF5文件
with h5py.File('test2/data/processed/FC_Grating_012.h5', 'r') as f:
    # 访问Spike数据
    spike_times = f['spikes']['spike_times'][:]
    spike_elec_ids = f['spikes']['spike_elec_ids'][:]
    
    # 访问试次信息
    trials = f['behavior']['trials'][:]
    
    # 访问LFP数据
    lfp_data = f['signals']['lfp_data'][:]
    channel_info = f['signals']['channel_info'][:]
```

2. 导入HDF5文件
3. 在Data Browser中浏览数据结构
4. 选择相应的数据进行分析

## 7. 结构设计特点

1. **层次化组织**：使用组和数据集的层次结构，清晰组织不同类型的数据
2. **复合数据类型**：使用复合数据类型存储结构化信息，如通道信息和试次信息
3. **高效存储**：使用适当的数据类型（如float32、int64）优化存储空间
4. **扩展性**：支持添加新的组和数据集，适应不同实验设计的需求
5. **互操作性**：标准HDF5格式，可被多种编程语言和工具访问

## 8. 应用场景

1. **Spike分析**：PSTH分析、栅格图、调谐曲线等
2. **LFP分析**：频谱分析、事件相关电位等
3. **行为分析**：ROC分析、反应时间分析等
4. **数据可视化**：各种图表和图形展示
5. **数据共享**：标准格式便于与其他研究人员共享数据

## 9. 总结

HDF5文件结构为NeuroPrime项目提供了一个高效、灵活的数据存储方案，能够满足神经电生理研究的各种数据需求。通过合理的层次化组织和数据类型选择，实现了数据的高效存储和快速访问，为后续的数据分析和可视化提供了坚实的基础。
