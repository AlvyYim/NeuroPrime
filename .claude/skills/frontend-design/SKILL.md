# Frontend Design Skill

This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

## Core Design Principles

### Aesthetic Direction
- **Visual Identity**: Create distinctive visual systems with clear typography, intentional color palettes, and cohesive design language
- **Depth & Atmosphere**: Add contextual depth through layered elements, subtle gradients, and purposeful shadows
- **Motion & Interaction**: Design meaningful animations that enhance user experience without being distracting
- **Typography Hierarchy**: Establish clear typographic scales and relationships
- **Color Psychology**: Use color intentionally to create mood and guide user attention

### Technical Implementation
- **Responsive Design**: Ensure layouts work across device sizes
- **Performance Optimization**: Prioritize fast loading and smooth interactions
- **Accessibility**: Implement proper contrast, keyboard navigation, and screen reader support
- **Code Quality**: Write clean, maintainable code with clear structure

## Usage Examples

### Basic Component
Create a login form with a modern, minimalist aesthetic. Include input fields, password visibility toggle, and a submit button with hover states.

### Animated Element
Design a navigation bar with smooth hover effects and a mobile-responsive hamburger menu that transforms into an X when opened.

### Complete Page
Build a dashboard page with card-based layout, data visualizations, and a responsive sidebar navigation.

## Design System Guidelines

### Color
- Establish a primary color palette with 3-5 core colors
- Define secondary and accent colors for specific elements
- Include light and dark mode variations
- Ensure sufficient contrast for accessibility

### Typography
- Select 2-3 complementary fonts (sans-serif for UI, serif for headings if appropriate)
- Define font sizes, weights, and line heights for different text levels
- Establish clear hierarchy between headings, body text, and UI elements

### Spacing
- Use a consistent spacing system (e.g., 4px base unit)
- Define spacing for different UI elements and components
- Maintain consistent margins and padding throughout the interface

### Components
- Buttons: primary, secondary, ghost, and icon variants
- Forms: input fields, checkboxes, radio buttons, select menus
- Cards: content containers with consistent styling
- Navigation: horizontal and vertical variants
- Modals: dialog boxes with proper overlay and animation

## Output Format

Provide complete, production-ready code with:
- HTML structure
- CSS styling (can use preprocessors like SCSS if specified)
- JavaScript for interactions (vanilla or framework-specific)
- Responsive design considerations
- Accessibility best practices

The code should be well-organized, properly commented, and ready for implementation.
